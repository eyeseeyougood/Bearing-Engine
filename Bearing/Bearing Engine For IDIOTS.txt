---- Common ideas ----

-- Threads --
The engine is single threaded (mostly)
This can often cause problems when you forget about how the threads work in Bearing Engine

The Tick Function on each component is infact single threaded, and so is rendering.
But the Delay() function of each component creates a new thread to invoke the Action provided
Due to parts of the engine like rendering being single threaded, the delay function and other
multi-threaded aspects of the engine can throw errors that make no sense and are practically
undebugable. Be very careful what you do inside the delay function and multiple threads in
general.
Other multi-threaded features include:
- events (rootLoaded, any events created by the user, etc)
